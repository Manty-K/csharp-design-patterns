# Creational Patterns

Creational patterns deal with object creation. They decouple the creation logic from the code that uses the objects, increasing flexibility and reuse.

| Pattern | One-liner |
|---|---|
| [Singleton](./singleton/) | Ensure a class has exactly one instance |
| [Builder](./builder/) | Construct a complex object step by step |
| [Factory](./factory/) | Create objects without specifying the exact class |
| [Abstract Factory](./abstract-factory/) | Create families of related objects |
| [Prototype](./prototype/) | Clone an existing object instead of building from scratch |
